# S3-compatible object storage

S3-compatible object storage is the **durability and economics tier**: it holds the Iceberg data
files, the index [backups](/product/functional/index-management/backup-restore.md), and the bytes
served for [cold-tier](/product/functional/cold-tiering.md) windows.

## How it's wired

Accessed via opendal (in-tree through iceberg-storage-opendal); configured with an endpoint +
credentials (`GROWLERDB_S3_ENDPOINT`, AWS keys). Any S3-compatible service works;
[MinIO](/system/runtime/dependencies/object-storage/minio.md) is the local/dev implementation.

## Notes

Object-storage economics — no always-on hot replicas of cold data — underpin the
[scale & cost](/product/non-functional/scale.md) NFR.
