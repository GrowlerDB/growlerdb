GrowlerDB

Open-source full-text search over Apache Iceberg

Iceberg stays the system of record. GrowlerDB maintains a fast, derived full-text index — search returns document keys that hydrate back to the authoritative rows in your lake.

Get started Documentation GitHub

Beta (0.x) — pre-1.0, production-tested, on the road to 1.0. AGPL-3.0.

1 · Index

Point at Iceberg

The connector streams your Iceberg changelog into a derived full-text index — no second copy to keep in sync by hand.

2 · Search

Query returns keys

Full-text, filters, ranges, aggregations. A search returns ranked document coordinates — not a stale copy of your data.

3 · Hydrate

Rows from the lake

Resolve keys back to the authoritative, governed rows in Iceberg by primary key. One source of truth.

Why GrowlerDB instead of Elasticsearch or OpenSearch?

  • No second copy of your data. Your lakehouse stays the system of record; the index is derived and rebuildable — not a parallel datastore to reconcile.
  • The lake is the source of truth. Hydration returns the live, catalog-governed Iceberg row, not a search-time snapshot that drifts.
  • No reindex-the-world migrations. Point at a table and go; the connector keeps the index current from the Iceberg changelog.
