GrowlerDB is installed as release "{{ .Release.Name }}" in namespace "{{ .Release.Namespace }}".

Topology (Design 14 sharded cluster):
  • control-plane  StatefulSet  {{ .Values.controlPlane.replicas }} replica(s)   — index registry (gRPC :{{ .Values.controlPlane.grpcPort }})
  • node           StatefulSet  {{ .Values.index.shards }} shard pod(s)   — one per shard ordinal of "{{ .Values.index.name }}", on PVs
  • gateway        Deployment   {{ if .Values.gateway.autoscaling.enabled }}{{ .Values.gateway.autoscaling.minReplicas }}–{{ .Values.gateway.autoscaling.maxReplicas }} (HPA){{ else }}{{ .Values.gateway.replicas }} replica(s){{ end }}   — public Engine API + console UI

Pod-K builds shard K of "{{ .Values.index.name }}" from `{{ .Values.index.sourceTable }}` on first boot, serves that
ordinal, and self-registers with the control plane; the gateway reads the live shard map from the
control plane and fronts each shard's primary (hot-reloading on change).

Watch it come up:
  kubectl -n {{ .Release.Namespace }} rollout status statefulset/{{ include "growlerdb.controlplane.fullname" . }}
  kubectl -n {{ .Release.Namespace }} rollout status statefulset/{{ include "growlerdb.node.fullname" . }}
  kubectl -n {{ .Release.Namespace }} rollout status deployment/{{ include "growlerdb.gateway.fullname" . }}

NOTE: this chart deploys control-plane + node + gateway. The index is partitioned into
{{ .Values.index.shards }} shard(s), one primary pod each. Zero-downtime per-shard *replicas* are a
follow-up (segment-shipping is single-shard only); HA today = shards spread across hosts + PDBs +
fast self-heal from the PV — a lost shard serves honest partial results until it recovers. The
streaming connector is an external Spark job, and compaction is in-process on the node.
